# 컴포넌트 아키텍처 & 리팩토링 가이드

본 문서는 코드의 유지보수성과 가독성을 극대화하기 위한 구조적 규칙입니다.

## 컴포넌트 설계 규칙

- 단일 책임 원칙: 하나의 컴포넌트는 하나의 명확한 역할만 수행해야 합니다.
- UI 컴포넌트 내부에는 비즈니스 로직(상태 관리, 데이터 페칭, 이벤트 핸들러 등)을 직접 작성하지 마세요.
- 모든 비즈니스 로직은 Custom Hook으로 분리하며, 파일명은 `[UI파일명]Hook.ts` 형식을 따릅니다.
- 공통으로 재사용 가능한 요소(버튼, 모달, 카드 등)는 개별 컴포넌트로 분리하여 `src/components/common`에 위치시킵니다.
